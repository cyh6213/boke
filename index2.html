Awareness 03

创新意识

Agent 不按照需求执行
❌ "检索竞品资料后生成分析报告" — Agent 直接生成，没去检索
❌ "先查天气再安排行程" — 查完天气直接给行程，忘了查交通
❌ "分析代码质量前先做安全扫描" — 跳过安全扫描直接分析
Plan Agent + 执行引擎解耦

Plan Agent 生成 DAG 图，执行引擎强制按图调度

🎤 你说
第三个是创新意识。我不满足于调 API 跑 Demo，每个项目都交出了自己的方案。 以 Workflow 项目为例—— 为什么做？使用 Agent 时发现它不按指令执行。要求它'检索后回答'，它直接生成答案。这不是策略不一致的问题，是 Agent 会无视你的指令，自己决定怎么做。 我的方案是什么？我设计了 Plan Agent + 执行引擎解耦的架构。Plan Agent 根据用户需求生成 DAG 图，执行引擎强制按图调度。Agent 在每个节点只能做该节点的事，不存在跳过某一步的可能。 核心能力有哪些？双层并行执行引擎、节点级断点恢复、Sub Agent 分发、三级异常治理、Plan 能力迭代优化。 👉 指着 Workflow 项目区域说"看这里"
