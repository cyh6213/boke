Project 01

企业知识库 Agentic RAG

项目来源 为什么做这个项目
核心优化 全链路优化
召回率优化 提升召回率（68% → 93%）
文本分块策略

两级四层语义分块：第一层父块流式解析（1MB 自动切割，避免 OOM），第二层子块按 512 Token 四级切割（段落 → 句子 → 词语 → 字符），全程保证语义完整性。引入重叠分块机制，保留上下文冗余。

代码文件使用 JavaParser 做 AST 分析，按类/方法提取，保持语义完整性。同时提取五种关系（extends / implements / imports / contains / calls）构建关系图谱。

问题改写策略

五种策略分层组合：简单改写（消除代词、补充隐含信息）、问题拆解（多问题分开查询）、HyDE（LLM 生成假设答案后检索）、多角度扩写（3-5 种表述分别检索后合并去重）、后退提问（先检索背景知识再细化）。

混合检索流程
1. 问题改写（不同问题不同策略）
2. 调用 Embedding 模型转 2048 维查询向量 + IK 分词
3. 向量检索（knn 余弦相似度），召回候选池
4. BM25 关键词检索，计算精确相关性分数
5. RRF 融合排序，合并两路结果
6. ReRank 精细排序，取前 k 篇
文件上传流程

前端按 5MB 分片上传 + MD5 校验。Redis Bitmap 防重复。所有分片上传完成后后端校验完整性，MinIO 合并，RocketMQ 异步触发解析分块向量化。

异常处理：幂等设计防重复上传、断点续传、分片重试、MD5 校验、定时任务核对 MinIO 与 MySQL 状态保证最终一致。

技术原理
向量检索原理

余弦相似度（只看方向）、欧式距离（看距离）、点积（方向+长度，需归一化）。长度代表信息量，角度代表语义。

BM25 原理

ES 默认关键词算法，k1 控制词频影响，b 控制长文档惩罚。

RRF 重排序

倒数打分 1/(k+rank)，忽略分数只看排名。k=60 平滑常数。

ReRank 原理

Embedding（Bi-Encoder）搜广度，Rerank（Cross-Encoder）做深度交互重排。

选型

Spring AI 框架 + 阿里 text-embedding-v4（MTEB 前三）+ ES / Milvus 存储 + BGE-Rerank / Cohere 重排。

幻觉优化 降低幻觉率（18% → 5%）
幻觉控制措施

Prompt 优化 + 结构化输出约束 + 低分拒答 + 幻觉检测机制，确保生成内容忠实于检索结果。

问答流程

用户提问 → 前置决策 → 问题改写 → 检索 → 低分时联网搜索兜底 → 记忆压缩 → 生成。

全链路双层内容审核：生成前风控拦截，生成后 RAGAS 校验忠实性，必要时多轮迭代补全。

智能路由 简单问题跳过检索

通过关键词/正则匹配对问题分类，简单问题（问候、数学等）直接回答，跳过向量检索，提升速度并降低 Embedding API 调用成本。

多跳推理 弥补全局性问题短板

针对复杂问题分解为子查询链，每步基于前序结论动态构建新查询。例如"程愚瀚和鲁迅的关系"→ 先查程愚瀚（电子科大学生）→ 再查电子科大的人才 → 关联到鲁迅。弥补向量 RAG 在全局性问题和多跳推理上的短板。

其他亮点
权限控制

基于 Spring Security 搭建三层过滤器链：

第一层：提取、校验 Token，封装用户信息到 SecurityContextHolder
第二层：按接口规则完成角色级权限校验
第三层：加载组织标签用于数据级权限控制
搜索模块自动构建带权限条件的 ES 查询，强制只返回用户有权查看的数据
MCP Server

实现 MCP Server，支持外部 AI Agent 通过标准化协议接入查询知识库，打破工具间的数据孤岛。
