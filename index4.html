企业知识库 Agentic RAG ↗


阶段一：基础建设与优化探索

第一阶段主要围绕企业知识库这个场景，搭建 RAG 的基础链路，学习并应用了各种常见的检索优化策略来提升召回率、降低幻觉率。同时基于企业知识库的实际需求，做了文档上传优化、权限控制等工程化的工作。

提高召回率 → 降低幻觉率 → 分片优化 → 问题改写 → 上传优化 → 权限控制 →

阶段二：将 RAG 变成了一个 Agentic RAG

第二阶段在基础链路上增加了两个核心能力。一个是智能路由，尝试对简单问题进行分流，减少不必要的向量检索；另一个是针对不同任务设计特定的检索策略，比如需要多次检索的问题、对比类的问题等，让 RAG 对不同类型的问题提供针对性的策略。这两个能力让 RAG 从被动检索变成了主动推理的 Agentic RAG。

智能路由 → 复杂问题策略分发 →

阶段三：建立测评体系

前两个阶段我其实没有系统的测评思路，所有的优化都是"看起来有提升就觉得有效"，但实际提升了多少、召回率和幻觉率到底怎么样，我并不清楚。所以在这个阶段，我系统性地搭建了测评体系，对检索和生成两个环节分别做了多轮消融实验，让每一次优化都能看到具体的数据变化。

测评数据 → RAG 测评文章 →

阶段四：对测评体系的进一步验证

在这个阶段我学到了非常关键的一点。在智能路由这里，我原本想通过关键词匹配拦截简单问题来跳过向量检索，从而节约时间。但实际测评下来发现这个方案有三个问题，第一检索管线只占总耗时的一小部分，第二简单问题的占比也不高，第三如果通过规则进行问题分类的准确度较低，会让不是简单问题的问题跳过检索，而如果使用模型来进行分类，虽然可以提高分类的准确率，但是耗时增加大于缩短，得不偿失。所以让我意识到：优化不是看起来合理就行，需要有测试数据验证。

可视化任务编排与执行系统 ↗
