# 面试演讲话术提示框制作指南

## 1. 核心概念

面试自我介绍页面采用**双内容结构**：
- **左栏（面试官可见）**：干净、结构化的视觉内容，像 PPT slide 一样——标题 + 关键数据 + 列表 + 表格
- **右下角固定话术框（仅你可见）**：你的演讲稿，滚动到不同区域自动切换内容

两者配合：面试官看左栏的结构，你读右下角的话术。

---

## 2. 技术实现

### 2.1 HTML 结构

```html
<!-- 每个 section 只需要 content-left，不需要内嵌 speech-right -->
<section id="hero" class="section-dual">
    <div class="content-left">
        <!-- 面试官看到的内容 -->
    </div>
</section>

<!-- 底部放一个固定话术框 -->
<div class="speech-fixed" id="speechFixed">
    <div class="speech-bubble">
        <div class="speech-label">🎤 你说</div>
        <div id="speechContent"></div>
    </div>
</div>
```

### 2.2 CSS 要点

```css
/* 固定定位到右下角，不挤压内容 */
.speech-fixed {
    position: fixed;
    right: 24px;
    bottom: 80px;           /* 避开 toggle 按钮 */
    width: 320px;
    z-index: 100;
    transition: opacity 0.3s, visibility 0.3s;
}
.speech-fixed.hidden {
    opacity: 0;
    visibility: hidden;
    pointer-events: none;
}

/* 话术框样式：紫色主题，和左栏区分 */
.speech-fixed .speech-bubble {
    background: #eef2ff;          /* 浅紫背景 */
    border-left: 4px solid #7c3aed;
    border-radius: 12px;
    padding: 16px 20px;
    font-size: 14px;
    color: #334155;
    line-height: 1.8;
    max-height: 60vh;             /* 防止太长溢出 */
    overflow-y: auto;
}

/* 响应式缩窄 */
@media (max-width: 1400px) { .speech-fixed { width: 260px; } }
@media (max-width: 1200px) { .speech-fixed { width: 220px; font-size: 13px; } }
@media (max-width: 900px)  { .speech-fixed { display: none; } }
```

### 2.3 JavaScript 滚动切换

```javascript
var speechContent = document.getElementById('speechContent');
var speechSections = [
    { id: 'hero', text: '话术内容1...' },
    { id: 'skills', text: '话术内容2...' },
    { id: 'workflow', text: '话术内容3...' },
];

function updateSpeechOnScroll() {
    var current = null;
    var scrollY = window.scrollY;

    for (var i = 0; i < speechSections.length; i++) {
        var el = document.getElementById(speechSections[i].id);
        if (el) {
            var top = el.offsetTop - 150;        // 偏移量 = 顶部导航高度 + 余量
            var bottom = top + el.offsetHeight;
            if (scrollY >= top && scrollY < bottom) {
                current = speechSections[i];
                break;
            }
        }
    }

    // 兜底：顶部显示第一个，底部显示最后一个
    if (!current) {
        var firstEl = document.getElementById(speechSections[0].id);
        if (firstEl && scrollY < firstEl.offsetTop - 150) {
            current = speechSections[0];
        } else {
            current = speechSections[speechSections.length - 1];
        }
    }

    if (current && speechContent) {
        speechContent.textContent = current.text;
    }
}

window.addEventListener('scroll', updateSpeechOnScroll);
window.addEventListener('load', updateSpeechOnScroll);
```

## 3. 话术内容写作规范

### 3.1 语言风格

| 场景 | 左栏（面试官看的） | 右下角话术（你讲的） |
|------|-------------------|---------------------|
| 工作流 | 术语准确、结构化 | 口语化、自然叙述 |
| 数据 | `recall_cap@5（上限截断召回率）` | "我做了系统的测试" |
| 能力 | 标题式列举 | "我设计了三级的异常治理..." |

**规则：**
- 话术用**第一人称**，像在跟面试官聊天
- 避免照读左栏内容，话术是**口述版本**
- 引入英文术语时加中文翻译，如"Reflexion（自反思）"
- 技术指标在左栏保留英文原名，话术中说中文意思

### 3.2 内容长度

- 每个区域的话术 **2-5 句话**
- 太短 → 信息量不够
- 太长 → 滚动太快来不及读
- 超出 60vh 自动滚动

### 3.3 叙述结构

好的话术按照"场景逻辑"组织，而非"页面结构"：

> **❌ 不好：** "这个页面有五个能力标签：并行执行、断点恢复..."
> **✅ 好：** "基于这个框架我实现了并行执行和断点恢复，通过 KAHN 拓扑排序让入度为 0 的节点并发跑..."

## 4. 常见问题排查

### 4.1 滚动切换不灵敏

- `offsetTop - 150` 的偏移量根据顶部导航高度调整
- 如果页面顶部有额外 padding，增加偏移量
- 在页面底部，最后一个 section 可能检测不到 → 加兜底逻辑

### 4.2 main.js 冲突

主站的 `main.js` 有 `updateActiveLink` 函数，会在滚动时移除所有 sidebar 的 `active` 类。如果项目页同时加载了 `main.js`，需要在 `updateActiveLink` 开头加：

```javascript
const sections = document.querySelectorAll('section[id]');
if (sections.length === 0) return; // 文章页/项目页没有 section，跳过
```

### 4.3 话术框重叠 toggle 按钮

- toggle 按钮 `bottom: 24px`
- 话术框 `bottom: 80px`（留出按钮空间）

## 5. 适用场景

| 场景 | 做法 |
|------|------|
| 自我介绍页 | 每个 section 一段话术，覆盖全过程 |
| 项目详情页 | 按 section id 分段，滚动到对应区域显示对应话术 |
| 文章页 | 一般不需要话术，除非用于演讲 |

## 6. 关键决策点

1. **固定 vs 跟随内容**：选 `position: fixed`，固定在右下角，不随内容滚动
2. **单个 vs 多个**：选单个话术框 + 内容切换，而不是每个 section 一个
3. **分段粒度**：一个 section 对应一段话术，不细分到子 section
