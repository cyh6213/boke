# 问答归档助手

## 使用场景
当用户提出有价值的技术问题并获得回答后，自动将问答内容归档到指定目录

## 归档规则
1. 过滤掉简单的确认性问题（如"好的"、"谢谢"等）
2. 只归档涉及以下领域的问题：
   - AI/大模型相关
   - 后端开发技术
   - Agent开发
   - 系统设计与架构
3. 问答对需要有实质性内容，不是闲聊

## 归档格式
采用「按类型分类 + 按时间命名」的混合存储方式：

**目录结构**：
```
docs/问答归档/
├── AI与Agent相关/
│   └── YYYY-MM-DD-问题关键词.md
├── 后端开发/
│   └── YYYY-MM-DD-问题关键词.md
├── 系统设计/
│   └── YYYY-MM-DD-问题关键词.md
└── 其他技术/
    └── YYYY-MM-DD-问题关键词.md
```

**分类规则**（严格遵循）：

| 分类目录 | 包含内容 | 示例问题 |
|---------|---------|---------|
| AI与Agent相关 | AI、大模型、Agent开发、RAG、LangChain、LLM、prompt工程 | "什么是RAG?"、"LangChain怎么用?"、"大模型原理" |
| 后端开发 | Java、Go、数据库、Redis、Spring、微服务、并发编程、消息队列 | "Java与Go的区别?"、"Redis缓存策略?"、"MySQL索引优化" |
| 系统设计 | 高并发、分布式、架构设计、系统优化、技术方案选型 | "如何设计秒杀系统?"、"分布式锁实现?"、"高可用架构设计" |
| 其他技术 | 不属于以上三类的其他技术问题 | "Git工作流?"、"Docker使用?"、"Linux命令" |

**文件名格式**：YYYY-MM-DD-中文关键词.md
**命名规则**：文件名必须使用中文，关键词部分为问题的核心中文描述，不能使用英文或拼音。

**内容结构**：

**单问答文件格式**：
```markdown
# 问题标题

## 问题
（面试官的问题）

## 回答
（专业、有深度的回答，包含技术选型思考）

## 引用来源
（回答中引用的外部资料来源）
```

**多问答追加格式**（在同一文件中追加新问答对）：
- 用分隔线 `---` 与前面内容分隔
- 新问答对使用 `## 问题x` 和 `## 回答x` 格式，x为序号

**回答内容格式**：
- 回答内容本身不要使用小标题（##）
- 但如果内容很长且有明显分区，可以在回答内部使用 `### 一、xxx`、`### 二、xxx` 形式的小标题做分区

## 归档位置
- 归档根目录：docs/问答归档/
- 自动根据问题类型创建子目录
