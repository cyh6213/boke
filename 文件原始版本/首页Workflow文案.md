# 首页 Workflow 项目介绍文案

## ① 从问题到方案

**问题：Agent 不按照需求执行**

使用 Agent 时发现指令明确但 Agent 不执行——比如要求检索后回答却跳过检索直接生成，让先做安全扫描再分析代码却直接跳过扫描。问题的核心是：Agent 会无视你的指令和规划，自己决定怎么做。

**两条确定化路径：** 我考虑了两种方案——DAG 约束"整个流程必须做什么"（如强制[读取财报→分析指标→生成报告]，无法跳过分析步骤），和 Hooks 约束"每一步前后必须做什么"（如 PreHook 检查文件格式、PostHook 检查输出是否含敏感数据）。DAG 对"Agent 跳过步骤"的问题更直接，因此选择了 DAG 路径。

**方案：** 将 DAG 图构建交给大模型来完成。Plan Agent 根据用户需求输出 DAG 数据（节点列表 + 边关系 JSON），不再参与执行；执行引擎按图自主调度、分发、并行运算。规划与执行彻底解耦，保证可见即所得。

## ② 核心能力

**1. 双层并行执行引擎**
第一层基于 KAHN 拓扑排序实现节点级并行——入度为 0 的节点直接并发执行，执行后更新下游入度，出现新的可执行节点继续并发。同时内置环路检测，通过跟踪入度变化识别死锁。第二层针对单节点内的工具调用实现细粒度并行（如同步读取多个文件时的并行执行），形成"节点级 + 工具级"双层并行。经 200 项任务评估集测试，相比串行基线，并行执行时间缩短 23%。

**2. 节点级断点恢复**
摒弃 LangGraph 整体状态快照方案，为每个节点独立持久化输入输出及状态至 MySQL（execution_snapshot 表）。运行时数据纯走内存传递，仅断点恢复时从快照重建上下文。支持按请求指定首个失败或未完成节点精准恢复，恢复前可手动修正节点输出或变量。相比 LangGraph 只能从最近 checkpoint 继续的方式，实现了更低存储开销与更灵活恢复粒度。

**3. Sub Agent 分发执行与 Token 优化**
执行引擎按职责将节点分发给三种 Sub Agent：调研 Agent（搜索类工具）、审查 Agent（代码检查类工具）、执行 Agent（命令执行/写文件类工具）。每类 Agent 只加载自身角色相关的 System Prompt 和工具描述，不加载全局规则和其他角色的工具。同时建立了公共工具池和私有工具池的划分。相比单 Agent ReAct，单次调用 System Prompt 从 9K 降至 3K（↓67%），且节点结束后上下文重置不跨节点累积，完整 3 步任务 Token 消耗降低 41%。

**4. 三级异常治理与条件分支路由**
固定 DAG 无法像 PAE 那样动态调整计划，因此设计了两个机制弥补。异常治理基于模板方法模式自上而下分三层：顶层 execute 封装重试策略（固定/线性/指数退避），中层 doExecuteWithTimeout 通过 AsyncUtil 做超时控制，底层 doExecute 统一兜底异常并按直接中断、错误码、失败分支三种策略分流。条件边分支让 Plan Agent 生成多分支 DAG，每条边带 condition 字段，节点执行后仅激活匹配路径上的边，通过级联裁剪未激活分支实现对 KAHN 调度器的零侵入式分支支持。

**5. SSE 实时推送**
在 Controller 层创建 SseEmitter，通过装饰器模式包装为 SseStreamCallback 传给工作流引擎，再包装为 WorkflowMsgCallback。执行事件放入 streamQueue 队列缓冲，后台线程异步消费并推送至前端，实现节点开始、结束、流式输出、异常等进度的实时展示，不阻塞执行引擎。

**6. Plan 能力迭代优化**
面对 Plan Agent 生成 DAG 图的不可靠问题，构建了从 Prompt 约束（JsonSchema 输出模板 + 拆解/生成/颗粒度规则）→ 后置校验（环路检测 + 步骤完整性 + 引用合法性）→ FewShot 样本（标准样本 + 反例样本）→ 分级 Plan（长任务粗分后子 Plan 独立生成）→ 双 Agent 评审（独立评审 Agent 输出修改意见）→ 迭代数据闭环（错误 DAG 自动归类并回流至样本库）的完整优化链路，DAG 图构建正确率达 97%。基于 Reflexion（NeurIPS 2023）的思想实现迭代闭环。节点分开存储天然保留每次执行的完整 Trace（输入、输出、工具调用、耗时、Token），结合 LLM-as-Judge 量化评估低分样本自动回流，形成持续的自改进循环。

## ③ 测评结果（保留，不变）

DAG 图构建正确率 97% · 子 Agent 任务完成率 94% · 并行执行时间缩短 23% · 单次调用 Token 降低 67%
